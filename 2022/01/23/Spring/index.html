BeanDefinition

BeanDefiniton是Spring中非常核心的概念。

正如字面意思，BeanDefiniton表示Bean定义。Spring根据BeanDefiniton来创建Bean对象。

BeanDefiniton中有很多的属性用来描述Bean。

BeanDefinition中的重要属性有

  1. beanClass

    表示一个Bean的类型，比如UserService.class, OrderServiece.class，Spring在创建Bean的过程中会根据此属性来实例化得到对象。

  2. scope

    表示一个Bean的作用域，比如：

    scope等于singleton，该bean就是一个单例Bean；scope等于prototype，该bean就是一个原型bean。

  3. isLazy

    表示一个Bean是不是懒加载。原型bean的isLazy属性不起作用；懒加载的单例bean，会在第一个getBean的时候生成该bean；非懒加载的单例bean，则会在Spring启动过程中直接生成好。

  4. dependsOn

    表示一个Bean在创建之间所依赖的所有Bean。在一个bean被创建前，它所依赖的所有bean必须先全部创建好。

  5. primary

    表示一个bean是主bean，在Spring中一个类型可以有多个bean对象。在进行依赖注入时，如果根据类型找到了多个bean，此时会判断这些bean中是否存在一个主bean，如果存在则直接将这个bean注入给属性。

  6. initMethodName

    表示一个bean的初始化方法，一个bean的生命周期过程中有一个初始化的步骤。Spring会在初始化步骤中调用bean的初始化方法，初始化逻辑由程序员自己定义，表示程序员可以自定义逻辑对bean进行加工。

平时所用到的@Component @Bean 标签都会解析为BeanDefinition对象。

BeanFactory

BeanFactory是一种“Spring”容器，BeanFactory根据翻译即为Bean工厂。

它可以用来创建bean，获取bean。

BeanFactory是Spring中非常核心的组件。

BeanFactory利用BeanDefiniton来生成Bean对象。BeanDefiniton相当于BeanFactory工厂的设计图，而Bean对象就是BeanFactory根据设计图生产出来的产品。

BeanFactory的核心子接口和实现类

  1. ListableBeanFactory
  2. ConfigurableBeanFactory
  3. AutoWiredCapableBeanFactory
  4. AbstractBeanFactory
  5. DefaultListableBeanFactory

DefaultListableBeanFactory是其中最重要的。它的功能有：支持单例bean、支持bean别名、支持父子BeanFactory、支持Bean类型转化、支持Bean后置处理、支持FactoryBean、支持自动装配，等等。

Bean生命周期

Bean生命周期描述的是Sring中一个Bean从 创建到销毁过程中所经历的步骤，其中Bean的创建过程是重点。

程序员可以利用Bean的生命周期对Bean进行自定义加工。

.png)

  1. 实例化

    通过构造方法反射得到一个实例化对象，在Spring中，可以通过BeanPostProcessor机制对实例化进行干预。

  2. 依赖注入

    实例化所得到的对象是不完整的对象，不完整的意思是该对象的某些属性还没有进行属性填充，也就是Spring还没有自动给某些属性赋值，属性填充就是我们通常所说的依赖注入、自动注入。

  3. 初始化

    在对一个对象进行依赖注入之后，Spring提供了初始化机制，程序员可以利用初始化机制来对Bean进行自定义加工。比如可以利用InitializingBean接口来对Bean中的其他属性进行赋值，或对Bean中的某些属性进行校验。

  4. 初始化后

    AOP机制就是在初始化后通过BeanPostProcessor机制实现的，初始化之后得到的对象就是我们所说的Bean对象。

@Autowired

@Autowired表示某个属性需要进行依赖注入，可以写在属性或方法上。注解中的require属性默认为true，表示如果没有对象可以注入给属性则抛出异常。

@Autowired加在某个属性上，Spring在进行Bean的生命周期过程中的依赖注入阶段会基于实例化出来的对象，对该对象中加了@Autowired标签的属性自动赋值。

Spring会先根据属性的类型(类名称)去Spring容器中找出该类型的所有Bean对象，如果找出来多个，再根据属性的名字(类的变量名)从多个中选择一个。如果required属性为true，并且根据属性名查找不到对象，则直接抛出异常；如果根据属性的类型只找出来一个Bean对象，则不再根据属性名字查找。（先byType，在byName）

注：完整的底层逻辑其实更加复杂，不仅仅只有这两步。

@Autowired加载某个方法上，Spring会在Bean生命周期的依赖注入阶段，根据方法的参数类型和参数名称从Spring容器中寻找Bean对象当作方法的入参传入，自动反射调用该方法。

@Autowired加在构造方法上，Spring会在推断构造方法阶段选择该构造方法进行实例化。在反射调用构造方法之前，会根据构造方法参数类型、参数名从Spring容器中找到Bean对象，当作构造方法入参。

@Resource

@Resource注解与@Autowired类似，也是用来进行依赖注入的。@Resource是java层面提供的注解，@Autowired是Spring所提供的注解，它们依赖注入的底层实现逻辑也不同。

@Resource注解中有一个name属性，针对name属性是否有值，@Resource的依赖注入底层流程也是不同的。@Resource的name属性如果有值，那么Spring会直接根据所指定的name值去Spring容器找Bean对象，如果找到了则成功（如@Resource(name = “str1”)），否则报错。@Resource中的name属性如果没有值，则Spring先判断属性的名字在Spring中是否有对应的Bean对象，如果有则成功注入，如果没有再根据属性的类型从Spirng容器中找Bean对象，找到则注入。（@Resouce查找bean的顺序是先byName，再byType。）

@Value

@Value注解也是对属性进行依赖注入的，只不过@Value属性是用来从Properties文件中来获取值的，并且@Value可以解析SpirngEL(Spring表达式)。

@Value(“ABC”) 这样的用法会直接将字符串”ABC”赋值给属性。如果属性不是String类型，或者无法进行类型转换，则报错。

@Value(\”${proper}\”) 这样的用法会将${}中的字符串当作key，从Properties文件中找出对应的key赋值给属性，如果没找到，则会把”${proper}”当作普通字符串注入给属性。在配置类中可以通过@PropertySource(classpath:spring.properties)将spring.properties文件导入（该文件放置在resourcess目录下）

@Value(“#{abc}”) 这样的用法会将#{}中的字符串当作Spring表达式进行解析。Spring会把”abc”当作beanName，并从Spring容器中找对应的bean，如果找到则注入，否则报错。

FactoryBean

FactoryBean是Spring所提供的的一种较灵活的创建Bean的方式，可以通过实现FactoryBean接口中的getObject()方法来返回一个对象，这个对象就是最终的Bean。

FactoryBean接口中有三个方法。

  1. Object getObject(): 返回的是Bean对象
  2. boolean isSingleton(): 返回的是是否是单例Bean
  3. Class getObjectType(): 返回的是Bean对象的类型

上述代码，实际上对应了两个Bean对象：

  1. beanName为”zhouyu”，bean对象为getObject()方法所返回的User对象。
  2. beanName为”&zhouyu”，bean对象为ZhouyuFactoryBean类的实例对象。

FactoryBean对象本身也是一个Bean，同时它相当于一个小型工厂，可以生产出另外的Bean。

BeanFactory是一个Spring容器，是一个大型的工厂，它可以生产出各种各样的Bean。

FactoryBean机制被广泛地应用在Spring内部和Spring与第三方框架或组件的整合过程中。

ApplicationContext

ApplicationContext是比BeanFactory更加强大的Spring容器，它既可以创建Bean、获取Bean，还支持国际化、事件广播、获取资源等BeanFactory不具备的功能。

ApplicationContext所继承的接口

  1. EnvironmentCapable

    ApplicationContext继承了该接口，表示拥有了获取环境变量的功能。可以通过ApplicationContext获取操作系统环境变量和JVM环境变量。

  2. ListableBeanFactory

    ApplicationContext继承了该接口，就拥有了获取所有beanNames、判断某个beanName是否存在beanDefiniton对象、统计beanDefiniton的个数、获取某个类型对应的所有beanNames等功能。

  3. HierarchicalBeanFactory

    ApplicationContext继承了该接口，就拥有了获取父BeanFactory、判断某个name是否存在bean对象的功能。

  4. MessageSource

    ApplicationContext继承了该接口，就拥有了国际化的功能。比如可以直接利用MessageSource对象获取某个国际化资源（比如不同国家语言所对应的字符）。

  5. ApplicationEventPublisher

    ApplicationContext继承了该接口，就拥有了事件发布的功能，可以发布事件。这是ApplicationContext相对于BeanFactory比较突出、常用的功能。

  6. ResourcePatternResolver

    ApplicationContext继承了该接口，就拥有了加载并获取资源的功能。这里的资源可是文件、图片或某个URL资源。

BeanPostProcessor

BeanPostProcessor是Spring所提供的一种扩展机制，可以利用该扩展机制对bean进行定制化加工。在Spring底层源码中广泛用到了该机制。BeanPostProcessor通常也叫做Bean后置处理器。

BeanPostProcessor是一个接口。我们定义一个后置处理器，就是提供一个类实现该接口。在Spring中还存在一些接口继承了BeanPostProcessor，这些子接口实在BeanPostProcessor的基础上增加了一些其他的功能。

BeanPostProcessor中的方法：

  1. postProcessBeforeInitialization()：初始化前方法，表示可以利用这个方法来对Bean在初始化前进行自定义加工。
  2. postProcessAfterInitialization()：初始化后方法，表示可以利用这个方法来对Bean在初始化后进行自定义加工。

InstantiationAwareBeanPostProcessor继承了BeanPostProcessor的一个子接口。它还多了一个方法：postProcessProperties()：属性注入后执行。

AOP

AOP就是面向切面编程，是一种非常适合在无需修改业务代码的前提下，对某个或某些业务增加统一的功能，比如日志记录、权限控制、事务管理等，能很好地解耦代码，提高开发效率。

AOP中的核心概念：

  1. Advice

    Advice即通知、建议。在Spring中通过定义Advice来定义代理逻辑。

  2. Pointcut

    Pointcut即切点，表示Advice对应的代理逻辑应用在哪个类、哪个方法上。

  3. Advisor

    Advisor=Advice+Pointcut，表示代理逻辑和切点的一个整体。程序员可以通过定义或封装一个Advisor，来定义切点和代理逻辑。

  4. Weaving

    Weaving表示织入，将Advice代理逻辑在源代码级别嵌入到切点的过程，就叫做织入。

  5. Target

    Target表示目标对象，也就是代理对象。在AOP生成的代理对象中会持有目标对象。

  6. Join Point

    Joint Point表示连接点，在Sping AOP中就是方法的执行电。

AOP是发生在Bean的生命周期过程中的：

  1. Spring在生成bean对象时，先实例化出来一个对象，也就是target对象。
  2. 在对target对象进行属性填充。
  3. 在初始化后步骤中，会判断target对象有没有对应的切面。
  4. 如果有切面，就表示当前target对象需要进行AOP。
  5. 通过Cglib或JDK动态代理机制生成一个代理对象，作为最终的bean对象。
  6. 代理对象中有一个target属性指向了target对象。

再谈@Bean

在前面章节中我们了解到@Bean可以写在配置类的方法上。即@Bean于@Configuration一起使用。然而这只是一种建议。因为添加@Configuration后，当调用@Bean标签方法时，不会真正去执行方法体，而是代理对象去执行，代理对象执行方法时会有额外的逻辑，如：判断spring容器中是否已经有想要获得的bean对象。

@Bean标签也可以写在@Component标志的类中，效果一致。

@Bean标签还可以写在自定义注解上。

@Bean(autowiredCandidate = false)

@Bean标签中的属性autowiredCandidate属性默认值为true，当将其设置为false，表示这个类不能进行自动注入。

@Conditional

@Conditional注解可以写在类或者方法上，表示其它注解生效的条件。如@Component注解和@Conditional注解添加在同一个类上，那这个类是否是@Component是有条件的。

@Conditional注解使用之前需要先实现Spring提供的一个Condition接口的实现类。

@Lazy

@Lazy标签可以加在类、字段、方法、参数上。都是利用的代理对象。

懒加载，懒加载的bean会在getBeany时生成，而不是启动Spring时生成。

@Lazy标签可以用来间接解决循环依赖问题。但是需注意在Spring中已经通过三级缓存解决了一部分循环依赖情况。
