MyBatis映射文件

MyBatis映射文件主要完成SQL语句的编写以及对结果集的映射。映射文件中的属性主要有：

  • select
  • insert
  • update
  • delete
  • sql

参数传递

MyBatis参数传递分为三种情况：

  1. 单个参数的传递
  2. 传递多个参数
  3. 传递封装参数

（1）单个参数传递：使用#{参数名}来引用，该参数名无需与接口中参数名相同，即可任意。

（2）多个参数传动：不能直接使用#{参数1}，#{参数2}引用。MyBatis将入参封装到了HashMap中。Map的默认键为”param1”，”param2”，……，”paramN”形式，因此可以使用#{param1}，#{param2}当作引用来传递。另外为了增强代码的可读性们可以在接口方法入参前使用@Param(“pName”)注解来为参数取名，从而可以在SQL中使用#{pName}获取属性名对应的参数。

（3）封装参数传递：封装参数有两种传递，一是直接传递POJO对象，一是传递Map。如果封装参数已经存在业务模型，则在SQL中直接使用对象定义中的属性名——#{属性名}引用即可。若不存在对应POJO则可以利用Map传递，Map中的键即为SQL中的索引名——#{键名}引用。

另外参数传递除了使用#{}形式，还可以使用${}形式。

  1. #{}采用占位符方式生成SQL，#{}方式下SQL在执行前参数位置以”?”形式显示。原生SQL中有些参数不允许占位，如表名、字段名。此时若要使用动态表名、字段名就需要使用${}。
  2. ${}采用拼接方式生成SQL，${}方式下参数位置直接为参数的值。因此${}方式存在SQL注入的风险。比如参数后拼接“;”再加上一个SQL语句，后者SQL语句就被执行。

select

resultMap

select输出结果需要使用resultType或者resultMap属性指明，且只能使用一个。

  1. resultType指定全类名或为全类名指定的别名。当返回集合时，需指定集合中的类型而不是集合本身。
  2. resultMap用于自动封装。resultMap可用于级连定义，即当返回结果中可以包含对象。而resultType做不到。
